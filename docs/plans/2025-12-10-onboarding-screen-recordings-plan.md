# Onboarding Screen Recording Scripts Plan

**Date:** December 10, 2025
**Purpose:** Comprehensive video documentation for employee onboarding from WordPress to Payload CMS
**Audience:** Non-technical content/marketing team members
**Format:** Hybrid approach with quick task videos (2-5 min) and workflow videos (8-12 min)
**Language:** English only
**Distribution:** YouTube (private/unlisted)

---

## Overview

This plan outlines 36 screen recording scripts organized into 10 sections, covering all frontend and backend features of
the Schoerke website. Videos follow a hybrid approach combining quick reference task videos with comprehensive workflow
demonstrations. Each recording shows the complete cycle: CMS editing → saving/publishing → frontend verification.

**Target Audience:**

- New employees learning the system for the first time
- Existing employees migrating from WordPress
- Non-technical content editors and marketing team

**Total Content:** ~2.5-3 hours across 36 videos

---

## Recording Structure

### Script Template

Each video script follows this standardized structure:

1. **Introduction (15-30 seconds)**
   - Video title and purpose
   - What you'll learn
   - Prerequisites (if any)

2. **Step-by-Step Walkthrough (main content)**
   - Clear numbered steps
   - Narration notes (what to say)
   - Screen actions (what to click/type)
   - Callout boxes/highlights for important UI elements

3. **Frontend Verification (30-60 seconds)**
   - Navigate to live site
   - Show where changes appear
   - Explain the connection between CMS and frontend

4. **Common Pitfalls (15-30 seconds)**
   - 1-2 mistakes to avoid
   - Quick troubleshooting tips

5. **Summary & Next Steps (15-30 seconds)**
   - Recap what was learned
   - Suggest related videos
   - Where to get help

---

## Production Guidelines

**Recording Environment:**

- **CMS:** Local development (localhost:3000/admin)
- **Frontend:** Live production site
- **Resolution:** 1080p minimum
- **Software:** Screen recording with cursor highlighting (Loom, ScreenFlow, Camtasia)

**Recording Best Practices:**

- Prepare test data before recording (sample artists, posts, etc.)
- Keep cursor movements smooth and deliberate
- Pause between major steps (easier editing)
- Use cursor highlights for important clicks
- Record narration separately if needed for clarity
- Clean up test data between takes

**Publishing:**

- Upload to YouTube as unlisted
- Use descriptive titles: "[Section #] - [Video Title]"
- Add video to organized playlist
- Include timestamps in description for longer videos

3. Prepare script outlines for first 5 videos
4. Choose and configure screen recording software

6. **Week 6:** Troubleshooting (3 videos) - Wrap up common issues

### Phase 3: Distribution

1. Create YouTube playlist organization
2. Upload all videos as unlisted
3. Add timestamps to video descriptions
4. Create master index document linking to all videos
5. Share playlist link with team
6. Gather feedback and iterate

### Phase 4: Maintenance

1. Update videos when features change
2. Track most-watched videos (indicates common needs)
3. Add new videos based on employee questions
4. Archive outdated videos
